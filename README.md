## What It Does

When this skill is active, your AI coding assistant (Claude Code / Cursor / Copilot) will:

1. **Only use Cake CSS variables** — `var(--fdn-color-palette-*)`, `var(--sem-color-*)`, never hard-coded hex
2. **Follow the fixed spacing scale** — `var(--fdn-spacing-space-{0..80})`, no arbitrary pixel values
3. **Apply correct border-radius** — `var(--fdn-radius-radius-{4,5,8,100,128})`
4. **Use proper shadows** — `var(--fdn-shadow-shadow-{1..5})`
5. **Match Figma variant structure** — Button(Filled/Outlined/Text × Default/Success/Error × sm/md)
6. **Reference component templates** — 29 components with pre-built React/Vue/HTML code

### Before (without skill)

```css
/* AI generates arbitrary values */
.button {
  background: #0F172A;
  color: white;
  padding: 10px 18px;
  border-radius: 6px;
}
```

### After (with skill)

```css
/* AI enforces Cake tokens */
.cake-btn--filled {
  background: var(--sem-color-bg-brand);
  color: var(--sem-color-text-inverse);
  padding: var(--fdn-spacing-space-8) var(--fdn-spacing-space-16);
  border-radius: var(--fdn-radius-radius-4);
}
```

## Token Architecture

The Cake Design System uses a **three-layer token architecture**:

```
┌─────────────────────────────────────────────────────────────┐
│  Foundation (--fdn-*)                                       │
│  33 colors · 15 spacings · 5 radii · 5 shadows · 39 fonts │
│  Raw design values from Figma                               │
├─────────────────────────────────────────────────────────────┤
│  Semantic (--sem-*)                                         │
│  26 semantic aliases                                        │
│  bg-brand, text-inverse, border-error, state-success...    │
├─────────────────────────────────────────────────────────────┤
│  Component (cmp.*)                                          │
│  260+ component-level tokens                                │
│  button.primary.bg, modal.shadow, table.row.hover...       │
└─────────────────────────────────────────────────────────────┘
```

### Key Semantic Tokens

| Variable | Hex | Usage |
|----------|-----|-------|
| `--sem-color-bg-brand` | #0F172A | Primary brand bg, filled buttons |
| `--sem-color-text-inverse` | #FFFFFF | Text on dark backgrounds |
| `--sem-color-interactive-primary-hover` | #1D4ED8 | Hover state |
| `--sem-color-border` | #737373 | Default borders |
| `--sem-color-border-error` | #B91C1C | Error borders |
| `--sem-color-state-success` | #F0FDF4 | Success bg |
| `--sem-color-state-error-light` | #FFF1F2 | Error bg |
| `--sem-color-bg-disabled` | #F5F5F5 | Disabled state |
| `--sem-color-text-disabled` | #737373 | Disabled text |

## Installation

### GitHub Copilot (Recommended — One-Click Install)

**方式 A：运行安装脚本（推荐）**

```powershell
# Windows PowerShell — 在你的项目根目录执行：
git clone https://github.com/zengwenju/ui-ux-lenovo-skill.git _temp_skill
powershell -File _temp_skill/scripts/install-copilot.ps1
Remove-Item _temp_skill -Recurse -Force
```

```bash
# macOS / Linux — 在你的项目根目录执行：
git clone https://github.com/zengwenju/ui-ux-lenovo-skill.git _temp_skill
bash _temp_skill/scripts/install-copilot.sh
rm -rf _temp_skill
```

安装后你的项目会自动获得：

## Usage

### Automatic Mode

Once installed, the skill activates automatically when you ask AI to build UI components:

```
Create a success button component
Build a data table with selection
Design a modal for confirming deletions
```

The AI will automatically use Cake tokens and reference the component templates.

### Search Engine (Advanced)

Use the built-in BM25 search engine to find exact token combinations:

```bash
# Search components
python scripts/search.py "button success filled" --domain component

# Search colors
python scripts/search.py "primary blue" --domain color

# Search typography
python scripts/search.py "heading bold 20" --domain typography

# Search spacing/radius/shadow
python scripts/search.py "card padding shadow" --domain spacing

# Auto-detect domain
python scripts/search.py "modal confirm"
```

## Anti-Patterns (Enforced)

| ❌ Don't | ✅ Do |
|----------|-------|
| `color: #0F172A` | `color: var(--fdn-color-palette-0f172a)` |
| `padding: 15px` | `padding: var(--fdn-spacing-space-16)` |
| `border-radius: 6px` | `border-radius: var(--fdn-radius-radius-8)` |
| `font-family: Arial` | `font-family: var(--fdn-typo-style-5-font-family)` |
| `box-shadow: 0 2px 4px rgba(...)` | `box-shadow: var(--fdn-shadow-shadow-1)` |
| Inventing arbitrary spacing | Use only the Cake spacing scale |
| Mixing other design systems | Use Cake tokens exclusively |
